# 终端服务部实验室 (lab-stack) 项目说明

## 项目概述

这是一个为终端服务部（Terminal Service Department）创建的实验室项目，旨在提供全面的技术学习资料、培训文档和知识管理系统。该项目专注于培养部门成员的技术能力，涵盖从基础技能到高级专业方向的完整学习路径。

项目核心目标：

- **Driving Infrastructure, Intelligence & Content** - 推动基础设施建设、智能化工具开发和内容创作
- 提供系统化的技术学习路径和培训材料
- 构建知识管理和智能体技术的实践平台

## 核心内容

### 1. 部门介绍 (doc/部门介绍.md)

详细介绍了终端服务部的职能、职责和学习方向，包括：

- 基础设施运维（服务器/NAS/门户）
- 知识管理与创新（智能体开发、知识库建设）
- 平台运营与内容创作（社交媒体运营、技术内容创作）

### 2. 学习路径 (doc/RoadMap.md)

采用树状学习路径结构，分为三个层次：

- **🟢 必要的基础知识** - 所有成员必须掌握的核心技能
- **🟡 建议学习的基础知识** - 推荐学习的技能，提升工作效率
- **🔵 拓展学习内容** - 专业深化内容，支持个人职业发展

### 3. 技术培训材料

- **VSCode 高效使用指南** - 从入门到精通的完整开发工作流
- **会议纪要** - 记录教学进度和学习成果
- **效率工具链** - 输入法、快捷键、效率工具等

## 学习方向覆盖

### 基础技术栈

- 计算机组成原理、网络基础
- Linux 服务器管理
- Git 版本控制
- Python 基础编程

### 前沿技术方向

- **现代 Web 开发** - React、Vue 等前端框架，全栈开发
- **APP 应用开发** - 跨平台框架、原生开发
- **大模型与智能体技术** - MCP 协议、A2A 通信、n8n 工作流
- **AI 创意与创作** - AIGC 工具、图像处理

### 效率提升工具

- 快捷键系统
- 效率工具链（Quicker、AutoHotkey）
- 知识管理（Notion）
- AI 辅助开发（VSCode Copilot、Cursor）

## 项目特点

### 1. 结构化学习路径

采用自由、多元的树状学习结构，允许成员根据个人兴趣和职业发展方向灵活选择学习内容。

### 2. 实践导向

注重理论与实践结合，每个知识点都强调动手实践和项目应用。

### 3. 持续更新

学习路径将根据技术发展和部门需求定期更新，确保内容的时效性和实用性。

### 4. 团队协作

支持团队协作学习，包含代码审查、技术分享、项目复盘等协作机制。

## 使用指南

### 新手成员

1. 从**必要的基础知识**开始，掌握计算机基础、Git、Linux基础
2. 学习VSCode高效使用方法
3. 参与部门基础工作，积累实践经验

### 进阶成员

1. 深入学习**建议学习的基础知识**，包括Web开发、Docker、AI基础
2. 选择1-2个专业方向重点发展
3. 参与实际项目开发

### 资深成员

1. 根据兴趣选择**拓展学习内容**
2. 在专业领域深入钻研
3. 指导新人，推动技术创新

## 技术栈与工具

### 开发环境

- **编辑器**: Visual Studio Code (推荐配置)
- **版本控制**: Git + GitHub
- **包管理**: 根据不同技术栈选择（npm, pip, cargo等）

### 云原生与运维

- **容器化**: Docker
- **云平台**: AWS/Azure/阿里云
- **监控**: Prometheus/Grafana

### AI与智能体

- **大模型**: 各种LLM平台
- **智能体开发**: n8n工作流编排
- **协议**: MCP协议、A2A通信
